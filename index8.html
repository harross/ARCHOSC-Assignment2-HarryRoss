Task 3 used a page table with a 32-bit virtual address where the top 20 bits identify the page and the lower 12 bits are the offset. Given a virtual address, we looked up the page frame from the first 8 entries of the table, then constructed the physical address by appending the original 12-bit offset to the retrieved page frame number.

For Task 4.2, the bonus task asked us to find the physical address for virtual address 0x1804. Decomposing with a 10-bit offset (lower order bits) and the remaining bits as the VPN:

Virtual address Binary VPN Offset
0x18040001 1000 0000 01006 (0x6)4 (0x004)
0x00FC0000 0000 1111 11000 (0x0)252 (0x0FC)

For 0x1804, the MMU takes VPN 6, checks the TLB, finds the corresponding PPN, then appends offset 4 to form the physical address. For 0x00FC (VPN = 0), if that page is non-resident (R = 0 in the page table), the MMU triggers a page fault and the OS must load the page from disk before translation can complete.

I then extended the example TLB class to track hits and misses, using a FIFO eviction policy when the 4-entry TLB is full:

public int? Lookup(int vpn) {
    foreach (var entry in entries) {
        if (entry.VPN == vpn) { Hits++; return entry.PPN; }
    }
    Misses++;
    return null;
}

public void Insert(int vpn, int ppn) {
    if (entries.Count >= capacity) entries.RemoveAt(0);
    entries.Add(new TLBEntry { VPN = vpn, PPN = ppn });
}

Running the access sequence 0,1,0,2,3,0,1,4,5,0,1,6,7,0,1 against the 4-entry TLB produced a 33.3% hit ratio - once the working set grows beyond 4 unique VPNs, entries 0 and 1 are repeatedly evicted and then missed on re-access:
